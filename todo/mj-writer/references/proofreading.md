# AI味审校

系统化降低文章AI检测率，增加人味的三遍审校专业能力。

## 何时使用

- 用户说"审校一下"、"降低AI味"、"太AI了"
- 初稿创作完成后
- 用户提到"AI检测率高"、"没人味"
- 文章需要最终polish

## 核心目标

- **去除AI味**：消除24种AI写作模式
- **注入MakerJackie风格**：按照写作风格指引主动改写，而不是只做减法
- **增加人味**：让文章有经验感、有温度、像真人在说话

---

## 三遍审校流程

### 第一遍：内容审校（逻辑、事实、结构）
**目标**：确保内容准确、逻辑清晰

**检查项**：
- [ ] 事实准确？（数据、时间、产品名称）
- [ ] 逻辑清晰？（前后无矛盾）
- [ ] 结构合理？（无跑题）
- [ ] 信息完整？（关键点都说了）
- [ ] 无编造？（所有数据和案例都是真实的）

**方法**：
- 从头到尾读一遍，标注疑问点
- 搜索验证不确定的信息
- 检查数字和专有名词的准确性

---

### 第二遍：AI味降重（核心）
**目标**：去掉AI味，增加人味

#### 核心规则速查

1. **删除填充短语** — 去除开场白和强调性拐杖词
2. **打破公式结构** — 避免二元对比、戏剧性分段、修辞性设置
3. **变化节奏** — 混合句子长度。两项优于三项。段落结尾要多样化
4. **信任读者** — 直接陈述事实，跳过软化、辩解和手把手引导
5. **删除金句** — 如果听起来像可引用的语句，重写它

#### 24种AI写作模式识别与改写

##### 内容模式（6类）

**1. 过度强调意义、遗产和更广泛的趋势**
- **特征词汇**：作为/充当、标志着、见证了、是……的体现/证明/提醒、极其重要的、核心的、关键性的作用/时刻、凸显了其重要性、反映了更广泛的、象征着其持续的、为……做出贡献、关键转折点、不断演变的格局
- **问题**：夸大任意事物的意义
- **改写方向**：直接陈述事实，不夸大意义

**2. 过度强调知名度和媒体报道**
- **特征词汇**：独立报道、地方/区域/国家媒体、由知名专家撰写、活跃的社交媒体账号
- **问题**：LLM反复强调知名度主张，不提供具体上下文
- **改写方向**：只提具体哪家媒体报道了什么事

**3. 以 -ing 结尾的肤浅分析**
- **特征词汇**：突出/强调……、确保……、反映/象征……、为……做出贡献、培养/促进……、展示……
- **问题**：在句子末尾添加分词短语增加虚假深度
- **改写方向**：去掉虚假深度，直接陈述

**4. 宣传和广告式语言**
- **特征词汇**：拥有（夸张用法）、充满活力的、丰富的（比喻）、深刻的、令人叹为观止的、必游之地、迷人的、坐落于、开创性的
- **问题**：LLM在中立话题上使用夸张的宣传语言
- **改写方向**：用中性语言

**5. 模糊归因和含糊措辞**
- **特征词汇**：行业报告显示、观察者指出、专家认为、一些批评者认为
- **问题**：将观点归因于模糊权威而不提供具体来源
- **改写方向**：直接说出具体来源

**6. 提纲式的"挑战与未来展望"**
- **特征词汇**：尽管其……面临若干挑战……、尽管存在这些挑战、未来展望
- **问题**：套用模板化的"挑战"段落
- **改写方向**：直接说具体问题

##### 语言和语法模式（6类）

**7. 过度使用的"AI词汇"**
- **高频词**：此外、与……保持一致、至关重要、深入探讨、强调、持久的、增强、培养、复杂/复杂性、关键、格局、展示、织锦、宝贵的、充满活力的
- **改写方向**：用日常词汇替代

**8. 避免使用"是"（系动词回避）**
- **特征词汇**：作为/代表/标志着/充当、拥有/设有/提供
- **改写方向**：直接用"是"、"有"

**9. 否定式排比**
- **特征结构**："不仅……而且……"、"这不仅仅是关于……，而是……"
- **改写方向**：简洁陈述

**10. 三段式法则过度使用**
- **问题**：强行将想法分成三组，显得套路化
- **改写方向**：检查是否自然，不必刻意避免或追求三这个数字

**11. 刻意换词（同义词循环）**
- **问题**：AI重复惩罚导致过度使用同义词
- **改写方向**：统一用同一词

**12. 虚假范围**
- **特征结构**："从X到Y" 结构，X和Y不在有意义尺度上
- **改写方向**：直接列举

##### 风格模式（6类）

**13. 破折号过度使用**
- **问题**：破折号（—）比人类更频繁
- **改写方向**：改为逗号或句号

**14. 粗体过度使用**
- **问题**：机械地用粗体强调短语
- **改写方向**：保留有意义的粗体（关键术语、核心结论），只删除不必要的机械重复

**15. 内联标题垂直列表**
- **问题**：列表项以粗体标题开头+冒号
- **改写方向**：改为散文

**16. 互联网黑话堆砌**
- **特征词汇**：赋能、闭环、颗粒度、底层逻辑、对齐、抓手、方法论、沉淀、倒逼
- **问题**：AI生成中文内容常套用这些空洞热词
- **改写方向**：换成大白话

**17. 表情符号**
- **问题**：用表情符号装饰标题或项目符号
- **改写方向**：删除

**18. 弯引号**
- **注**：中文排版应使用中文全角引号（“ ”），不适用此规则

##### 交流模式（3类）

**19. 协作交流痕迹**
- **特征词汇**：希望这对您有帮助、当然！、您说得完全正确！、请告诉我
- **改写方向**：直接陈述事实

**20. 知识截止日期免责声明**
- **特征词汇**：截至[日期]、根据我最后的训练更新、虽然具体细节有限
- **改写方向**：直接陈述已知事实

**21. 谄媚/卑躬屈膝的语气**
- **问题**：过于积极讨好的语言
- **改写方向**：改为简单陈述

##### 填充词和回避（3类）

**22. 填充短语**
- "为了实现这一目标"→"为了实现这一点"、"值得注意的是数据显示"→"数据显示"
- "由于下雨的事实"→"因为下雨"、"在这个时间点"→"现在"
- "系统具有处理的能力"→"系统可以处理"

**23. 过度限定**
- "可以潜在地可能被认为…"→"可能会影响结果"

**24. 通用积极结论**
- "公司的未来看起来光明"→"公司计划明年再开两个地点"

#### 具体改写方法

1. **逐段检查**，标注AI味表达
2. **对照上述24种模式清单** 逐一改写
3. **对照MakerJackie写作风格指引** 主动改写（不只是去掉坏的，要写成对的）
4. **加入个人经历和真实细节**
4. **短期自查 checklist**:

- ✓ 连续三个句子长度相同？打断其中一个
- ✓ 段落以简洁的单行结尾？变换结尾方式
- ✓ 揭示前有破折号？删除它
- ✓ 解释隐喻或比喻？相信读者能理解
- ✓ 使用了"此外""然而"等连接词？考虑删除
- ✓ 三段式列举？检查是否自然而非刻意为之

#### 常用改写技巧

- 长句拆短句（超过30字的拆成2-3句）
- 抽象改具体（"很多" -> "3个"，"显著提升" -> "省3-5小时"）
- 书面改口语（"进行操作" -> "点击"）
- 加入真实感受（"我之前用的时候……"、"说实话……"）

#### 常见错误与改写示例

**错误1：开头绕圈子**
```
❌ "在当今AI技术飞速发展的背景下，编程工具领域也迎来了新的变革。作为一款新兴的AI编程助手，Claude Code正在受到越来越多开发者的关注……"

✅ "Claude Code出了。我用了两周，确实比Cursor好用。今天聊聊为什么。"
```

**错误2：过度使用AI句式**
```
❌ "Claude Code不是简单的代码补全工具，而是一个智能编程助手。它不仅能够生成代码，而且能够理解开发者的意图。"

✅ "Claude Code不只是补全代码。它能理解你想干嘛。比如你说'重构这个模块'，它就知道该怎么做。"
```

**错误3：缺少具体细节**
```
❌ "经过一段时间的测试，我发现Claude Code在很多方面都表现优秀。"

✅ "用了两周。最明显的变化是，之前要3天的项目，现在1天半就能搞定。"
```

**错误4：态度过于中立**
```
❌ "Claude Code有其优势，也存在一些不足，具体是否适合使用需要根据实际情况判断。"

✅ "Claude Code确实好用。但价格贵，个人开发者可能扛不住。如果你公司报销，强烈推荐。"
```

### 第三遍：细节打磨（标点、排版、节奏）
**目标**：让文章读起来舒服、自然

**检查项**：
- [ ] 句子长度合适？（15-25字为主，不超过30字）
- [ ] 段落不太长？（手机屏幕3-5行）
- [ ] 标点自然？（多用句号，少用逗号连接长句）
- [ ] 节奏有变化？（快慢结合）
- [ ] 小标题不密集？（每300-500字一个）
- [ ] 列表适度？（能用散文就不用列表）
- [ ] 加粗适度？（不要过度，只在关键处使用）

**方法**：
- 大声朗读，感受节奏
- 找出超过30字的长句，拆短
- 检查段落长度，过长的分段
- 调整排版，删除过度的格式化

## MakerJackie写作风格指引

去掉AI味之后，还要让文字读起来像同一个人在说话。以下是MakerJackie文章里反复出现的风格特征。

**注意看场合**：个人观点、故事类的文章可以多带点个人表达；技术教程以把事说清楚为主，风格收着点。

### 核心规则

**1. 用"我"，不用"我们"**
直接说自己看到的、做到的。不需要"我们发现"、"人们往往"。
> 这周用 OpenClaw 搭了个自动化流程。

**2. 开头别绕**
第一句就说要聊什么。不要"在当今AI飞速发展的背景下"。
> 做社区一年，我陷入了三个困境。

**3. 有数字，不模糊**
"很多"→具体的数字，"显著提升"→省了多长时间。
> 之前要 3 天的项目，现在 1 天半能搞定。

**4. 短段短句，手机上看得舒服**
一段手机屏 3-5 行。一句话可以自成一段。大段文字拆开。

**5. 说人话**
"进行操作"→"点击"。"赋能"、"对齐"、"闭环"这些词删掉。

**6. 有态度**
不要说"有利有弊需根据实际情况判断"。好就是好，不好就是不好。
> Claude Code 确实好用。但贵。

### 加分项（个人观点类文章可以用）

**7. 偶尔暴露真实感受**
说"说实话，这一步对我来说挺不容易"可以，但不要刻意煽情。

**8. 口语化但不过头**
"先泼两盆冷水"、"坦诚讲"这种自然的可以。"妈蛋"、"瞎几把"只偶尔用。

**9. 用括号补充**
括号 = 悄悄话。放点背景信息或吐槽。

### 改写示例

以下示例展示的是手法的变化，不是让你编故事。

**示例1**
AI原文：
> 在当今快速发展的数字化时代，AI技术正在深刻改变着我们的工作方式。作为一种创新的编程工具，Claude Code为开发者提供了更加高效的解决方案。它不仅能够生成代码，而且能够理解上下文，从而显著提升开发效率。

→
> Claude Code 不只是补全代码。它能理解上下文。

删了什么：万能开场白、"不仅……而且……"句式、空洞的"高效解决方案"

**示例2**
AI原文：
> 做产品不是先找一个看起来热门的点子，而是先确认这件事是否适合你。

→
> 做产品，先别管热不热门。这事你愿不愿意干很久？

删了什么："不是……而是……"结构。"热门"比"看起来热门的点子"直接。

**示例3**
AI原文：
> 如果你不是按主线从头读，可以先按目标选入口。

→
> 不想从头看？直接跳到你要的那节。

删了什么："如果……可以……"条件句式。反问比条件句有力。

**示例4**
AI原文：
> AI 编程工具很多，但新手别一开始就全装一遍。先选一个，用它完成一个真实项目，比同时研究十个工具更有用。

→
> AI 编程工具别贪多。先选一个，拿它做个真实项目再说。

删了什么：第三人称说教（"新手"）、"但"转折、"比……更……"比较框架。建议简短直接就行，不需要编个人经历来佐证。

**原则**：优先做减法（删套路），而不是做加法（编故事）。宁可朴素，不要虚假。

改写完成后，从5个维度评分（1-10分，总分/50）：

| 维度 | 评估标准 | 得分 |
|------|----------|------|
| **直接性** | 直接陈述事实还是绕圈宣告？10分：直截了当；1分：充满铺垫 | /10 |
| **节奏** | 句子长度是否变化？10分：长短交错；1分：机械重复 | /10 |
| **信任度** | 是否尊重读者智慧？10分：简洁明了；1分：过度解释 | /10 |
| **真实性** | 听起来像真人说话吗？10分：自然流畅；1分：机械生硬 | /10 |
| **精炼度** | 还有可删减的内容吗？10分：无冗余；1分：大量废话 | /10 |
| **总分** | | **/50** |

**标准**：
- 45-50：优秀，已去除AI痕迹
- 35-44：良好，仍有改进空间
- 低于35：需要重新修订

## 改写示例

**改写前（AI味道）**：
> 新的软件更新作为公司致力于创新的证明。此外，它提供了无缝、直观和强大的用户体验——确保用户能够高效地完成目标。这不仅仅是一次更新，而是我们思考生产力方式的革命。行业专家认为这将对整个行业产生持久影响，彰显了公司在不断演变的技术格局中的关键作用。

**改写后（人性化）**：
> 软件更新添加了批处理、键盘快捷键和离线模式。来自测试用户的早期反馈是积极的，大多数报告任务完成速度更快。

**所做更改**：
- 删除了"作为……的证明"（夸大的象征意义）
- 删除了"此外"（AI词汇）
- 删除了"无缝、直观和强大"（三段式法则+宣传性语言）
- 删除了破折号和"确保"短语（肤浅分析）
- 删除了"这不仅仅是……而是……"（否定式排比）
- 删除了"行业专家认为"（模糊归因）
- 删除了"关键作用"和"不断演变的格局"（AI词汇）
- 添加了具体功能和具体反馈
