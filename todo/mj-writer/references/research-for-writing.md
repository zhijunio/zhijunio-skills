# 调研与信息搜索 Skill

结构化的网络调研与多渠道信息搜索流程，核心目标：调研成果实时持久化，防止会话截断丢失工作，通过交叉验证确保信息可靠性。

## 何时使用

- 为写文章做前期调研
- 了解新产品、新技术、新发布
- 搜集竞品信息或行业动态
- 涉及新产品/新技术（3个月内发布）
- 需要最新信息验证
- 任何需要多次 WebSearch 的信息搜索任务

## 核心流程

### Step 1: 立即创建调研文件

在开始搜索之前，先创建文件，确保第一次搜索结果就有地方保存。

路径：`<project-knowledge-base>/research-<主题>-<YYYYMMDD>.md`

初始内容包含：调研目标、关键问题、预期输出

```markdown
# [主题] 调研笔记

调研日期：YYYY-MM-DD
调研目标：[一句话说明]

## 关键问题
1. [问题1]
2. [问题2]
3. [问题3]

## 发现

（调研中逐步填充）

## 来源列表

（每次搜索后追加）
```

### Step 2: 多渠道搜索与增量保存

遵循以下搜索策略，每次 WebSearch 后立即将发现追加到文件。

#### 搜索内容
- 产品/技术的官方信息（发布时间、特性、定价）
- 科技媒体报道（TechCrunch, The Verge, Medium等）
- 社区讨论（Reddit, Hacker News， X/Twitter）
- 竞品对比信息
- 相关概念解释

#### 信息源优先级

**优先采信（半年内）**：
1. 权威科技媒体（TechCrunch, The Verge, MIT Tech Review）
2. 社区论坛（Reddit, Hacker News）
3. 官方文档和研究

**完全忽略**：
- 知乎、百度：2025年之前的信息全部忽略
- 过时内容：超过半年（6个月）
- 营销软文

#### 关键词组合策略
- 主题 + 年份：`"AI detection 2025"`
- 主题 + best practice：`"AI text humanization best practices"`
- 主题 + Reddit：`"AI writing detection reddit 2025"`

#### 增量保存格式

每次搜索后将发现追加到文件中，每条发现附上来源 URL 和日期：

```markdown
### 发现：<搜索主题>

- [发现1]（来源：[URL](链接)，日期：YYYY-MM-DD）
- [发现2]（来源：[URL](链接)，日期：YYYY-MM-DD）
```

### Step 3: 阶段摘要

每完成3次搜索，在文件中保存一次阶段摘要：

```
### 阶段摘要（第N轮）

本轮关键发现：
1. ...
2. ...

交叉验证结论：...
待验证点：...
```

### Step 4: 最终简报

调研结束时，整理文件为结构化简报：

```markdown
## 调研结论

### 关键事实
1. [事实1]（来源：URL）
2. [事实2]（来源：URL）

### 来源列表
| 来源 | URL | 发布日期 | 可信度 |
|------|-----|---------|--------|
| ... | ... | ... | 高/中/低 |

### 待确认问题
- [还需要进一步验证的点]

### 写作建议
- [基于调研结果，对后续写作的建议]
```

## 知识管理

### 保存格式

文件命名：`<project-knowledge-base>/<分类>/<主题>-<YYYYMM>.md`

```markdown
# [主题] 相关信息

**信息收集时间**：YYYY-MM-DD
**下次更新建议**：YYYY-MM-DD

## 核心发现
- 发现1
- 发现2

## 数据和链接
- 官方文档：xxx
- 媒体报道：xxx

## 信息来源
1. [来源1](链接)
2. [来源2](链接)
```

### 保存位置

- 调研笔记：`<project-knowledge-base>/research-<主题>-<YYYYMMDD>.md`
- 长期知识：`<project-knowledge-base>/<分类>/<主题>-<YYYYMM>.md`

其中 `<project-knowledge-base>` 根据当前项目配置，可设置为项目根目录下的 `_knowledge_base/` 或项目指定的其他知识库路径。

## 关键原则

- **先建文件再搜索**：确保第一次搜索结果就被保存
- **增量保存，不等到最后**：每次搜索后立即追加
- **调研和写作分离**：本 Skill 只做调研，不开始写草稿
- **标注来源与可信度**：区分一手信息（官方）和二手信息（媒体/社区），所有信息必须标注来源和时间
- **交叉验证**：多渠道对比验证，信息冲突时优先采信权威来源
- **忽略过时信息源**：知乎/百度（2025年前）、营销软文
- **不确定的信息向用户说明**
- **设定下次更新建议时间**，便于知识库维护

## 与其他 Skill 的关系

- 调研完成后，用户可触发 `mj-writer` 来确定写作方向并产出内容
- 调研文件将作为后续写作的输入素材
- 如果调研中发现的信息适合长期留存，保存到对应的知识库分类目录
