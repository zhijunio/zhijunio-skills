# 技术教程写作指南

> 面向不懂技术的产品经理和普通小白，想用 AI 工具做产品的人

## 核心原则

### 1. 说人话，不说术语

**能替换就替换：**
- ❌ Custom Hostname 机制 → ✅ 让 Cloudflare 认识你的网站
- ❌ DNS 层入口调度 → ✅ 帮你找更快的入口
- ❌ Fallback Origin → ✅ 备用地址
- ❌ 301 跳转 → ✅ 自动跳转

**必须用术语时立即解释：**
- 用类比："备用地址就像保险"
- 用场景："这个功能让 Cloudflare 认出这是你的网站"

### 2. 结构：先简后详

```
开头：一句话说清楚问题（用具体场景，不要抽象）
  ↓
问题分析：2-3 段话快速说清楚为什么（点到为止）
  ↓
方案对比：快速列出所有方案（每个一句话 + 标注"适合谁"）
  ↓
详细配置：每个方案单独一节（这里才展开步骤）
```

**示例：**

```markdown
## 问题到底是什么

你的网站部署在 Cloudflare 上，国内访问慢...

## 为什么会这样

简单说，问题出在两个地方：...

## 3 个方案快速对比

### 方案 1：用 AxisNow（推荐）
帮你在国内找更快的 Cloudflare 入口。
**适合**：想要省心方案的人。

### 方案 2：自己配置 DNS
自己控制流量走向。
**适合**：喜欢自己掌控的人。

## 方案 1：详细配置

### 核心思路
...

### 配置步骤
...
```

### 3. 步骤说明：三段式

每个配置步骤包含：

**在哪里操作 + 要做什么 + 为什么**

```markdown
#### 步骤 1：开启"识别其他入口"功能

在 Cloudflare 控制台：

```
选择域名 01mvp.com → SSL/TLS → Custom Hostnames → 启用
```

正常情况下，Cloudflare 只认识你直接添加到账号的域名。开启这个功能后，即使请求从其他入口（比如 AxisNow）进来，Cloudflare 也能认出这是你的网站。
```

**关键点：**
- "为什么"要解释清楚：不做这一步会怎样？做了之后有什么效果？
- 如果有特殊情况，单独说明

### 4. 用具体示例，不用占位符

**全文统一用一个真实示例：**
- ✅ 01mvp.com
- ❌ example.com
- ❌ your-domain.com

让读者能直接复制粘贴（只需要改域名）。

### 5. 说明局限性和风险

**必须说明：**
- 这个方案的缺点
- 什么情况下不适合用
- 依赖的服务是否稳定
- 需要持续维护的部分

```markdown
**注意：** AxisNow 这个方案理论上可行，但目前服务不太稳定，建议暂时观望。

**问题：**
1. **依赖第三方免费服务**：可能随时停止维护
2. **稳定性无法保证**：免费服务的可用性和速度都不稳定
3. **是个 trick**：利用了 Cloudflare 的机制，不一定长期有效

**适合谁：**想快速测试效果，但不适合生产环境。
```

### 6. 语气：简洁直接

**去掉铺垫词：**
- ❌ "简单说，就是..."
- ❌ "你可能注意到了..."
- ❌ "这一步做完，Cloudflare 就能认出你的域名了。"
- ✅ "AxisNow 帮你在国内找更快的 Cloudflare 入口。"
- ✅ "注意：我们用的是..."
- ✅ "这一步让 Cloudflare 认识你的域名。"

**不要用对话式语气：**
- ❌ "这是干嘛用的？"
- ✅ 直接陈述事实

### 7. 代码块和命令格式

**命令：**
```bash
pnpm install
```

确保读者可以直接复制。

## 常见问题

**Q: 什么时候需要解释"为什么"？**

每个配置步骤都要解释。读者需要知道：
- 不做这一步会怎样
- 做了之后有什么效果
- 这一步在整个流程中的作用

**Q: 如何判断是否太啰嗦？**

问自己：
- 这句话去掉后，读者还能理解吗？
- 这个词去掉后，意思会变吗？
- 如果答案是"能理解"、"不会变"，就删掉

**Q: 如何平衡"说人话"和"技术准确性"？**

- 优先说人话
- 如果术语是必须的（比如在 UI 里就叫这个名字），就用术语，但立即解释
- 可以用"也叫..."的方式同时给出术语和通俗说法

## 示例对比

### 改进前

```markdown
### Cloudflare 配置步骤

#### 第 1 步：启用 Cloudflare for SaaS

**在哪里操作：** Cloudflare 控制台 → 选择你的域名 → SSL/TLS → Custom Hostnames

**要做什么：** 点击启用 Cloudflare for SaaS

**这是在干嘛？**

正常情况下，Cloudflare 只认识直接添加到你账号里的域名。但现在我们要让请求从 AxisNow 绕一圈再进来，Cloudflare 就不认识了。

Cloudflare for SaaS 就是告诉 Cloudflare："嘿，如果有人用别的入口访问我的域名，你也要认出来这是我的站。"
```

### 改进后

```markdown
### Cloudflare 配置步骤

#### 步骤 1：开启"识别其他入口"功能

在 Cloudflare 控制台：

```
选择域名 01mvp.com → SSL/TLS → Custom Hostnames → 启用
```

正常情况下，Cloudflare 只认识你直接添加到账号的域名。开启这个功能后，即使请求从其他入口（比如 AxisNow）进来，Cloudflare 也能认出这是你的网站。
```

**改进点：**
1. 标题从术语改成通俗说法
2. 去掉了小标题，直接陈述
3. 路径导航改成更清晰的格式
4. 去掉了对话式的语气
5. 用更简洁的方式说明作用
